2. Dual-Memory System Architecture UPDATED

AURA uses a hybrid dual-memory approach:
① Long-term (ChromaDB): Every interaction is vectorised and stored forever in aura_memory.db. On each new query, AURA retrieves the 15 most semantically similar past turns, then a deduplication filter keeps only unique documents before injecting them into the prompt — ensuring personal details like the user's name are never missed.
② Short-term (Sliding Window): A Python list capped at MAX_HISTORY_TURNS=20 turns (40 messages). Resets only on New Chat or server restart.
