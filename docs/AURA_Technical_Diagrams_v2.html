Resilient LLM Cascade

Hierarchical failover logic ensuring 99.9% uptime.

flowchart TD Req([Incoming Request]) --> NIM[1. NVIDIA NIM: LLaMA-3 70B] NIM -- Success --> Resp([Response Object Generated]) NIM -- Timeout / 500 Error --> OR[2. OpenRouter: 4o-mini] OR -- Success --> Resp OR -- Timeout / 500 Error --> Gem[3. Gemini: Flash 2.0] Gem --> Resp
Technical Description: This flowchart validates our resilient cascade algorithm. The logic first attempts inference via NVIDIA NIM. If interrupted, the request transparently proxies to OpenRouter and then to Google Gemini, shielding the User from experiencing any "API down" failures.
Perception

Multimodal Emotion Fusion

Fusing discrete API signals into one cohesive emotional state.

flowchart LR A[Vocal Audio] -->|Wav2Vec2 Inference| F((FUSION ALGORITHM)) B[Text Transcript] -->|RoBERTa Inference| F C[Webcam Image] -->|Face-API.js| F F -->|Threshold Selection| E[Final Empathic State] F -.->|Conflict Triggered| R[Fallback Prompt injected]
Technical Description: Represents the Triple-Signal Fusion logic. Signals are processed in a strict hierarchical order:
  • P1: Audio Tone (Tone-of-Voice) — Detected via Wav2Vec2. Primary truth.
  • P2: Text Sentiment (Words) — Detected via DistilRoBERTa NLP. Secondary truth.
  • P3: Visual Face (Camera) — Detected via Face-API.js. Backup signal.
If a contradiction is detected (e.g. laughing while saying "I am sad"), the Fusion handler activates local override prompts.

Technical Description: Visualizes the blendshape attenuation process. Output weights generated by NVIDIA ACE for `mouthStretch` and `mouthPucker` occasionally distort the topology. The Server enforces a strict 75% dampening cap to maintain perfect facial structure irrespective of volume.
